# 设计: 交互方案

基于场景文档，设计 AI 助手的具体交互。核心原则：**极简**。

---

## 整体布局

保持现有双面板不变，改动集中在右侧面板的内容呈现和 AI 行为上。

```
┌─────────────────────────────────────────────────────────┐
│  会议 AI 助手          [观察/顾问/研究员]  ● 录音中 32:15 │
├────────────────────────┬────────────────────────────────┤
│                        │                                │
│                        │   AI 信息流                    │
│   实时转写              │   （卡片时间线，自上而下）      │
│   （和现在一样）        │                                │
│                        │                                │
│                        │                                │
│                        ├────────────────────────────────┤
│                        │ [输入框...]              [发送] │
└────────────────────────┴────────────────────────────────┘
```

右侧面板从"聊天窗口"改为**信息流**。区别不大，但心智模型变了：不是"我和 AI 聊天"，而是"AI 在旁边递纸条，我偶尔回一句"。

---

## 信息流

### 卡片，不是聊天气泡

现在的 ChatBubble 是对话气泡（系统/用户/AI 三色），暗示一来一回的对话。改为**卡片**，每张卡片是 AI 的一次独立输出，视觉上更像便签而非聊天。

卡片只有三种，对应场景中提炼的三种 AI 输出模式：

### ① 主动洞察（AI 自己想说的）

AI 分析转写后主动输出的内容：新角度、质疑、提问、联想。

```
┌──────────────────────────────────────────┐
│ 14:32                                    │
│                                          │
│ 都倾向方案 A 了，但 WebSocket 在弱网下    │
│ 维护连接成本很高。你们的用户有多少是       │
│ 移动端的？如果超过 30%，运维复杂度可能     │
│ 被低估了。                                │
│                                     [📌] │
└──────────────────────────────────────────┘
```

- 没有"💡"、"🤔"之类的类型标签。AI 说什么就是什么，不需要分类
- 右下角一个 📌（钉住）按钮——点了这张卡片会被标记，散会后方便回顾
- 不点也行，大多数时候不会点

### ② 应答（用户提问后的回复）

用户在输入框提问后，AI 的回复。和主动洞察在视觉上的唯一区别：上面会显示用户的原始提问（灰色小字）。

```
┌──────────────────────────────────────────┐
│ 14:36                                    │
│ ▸ 你问: PLG 是什么                        │
│                                          │
│ PLG = Product-Led Growth，让用户免费试用  │
│ 后自然转化付费。跟你们现在销售驱动模式    │
│ 比，前期投入在产品体验上，获客成本低但    │
│ 转化周期更长。                            │
│                                     [📌] │
└──────────────────────────────────────────┘
```

### ③ 节点总结（话题切换或会议结束时）

视觉上和其他卡片的区别：有一条分割线和标题"阶段小结"。

```
┌──────────────────────────────────────────┐
│ ── 阶段小结 ──────────────────── 14:50 ──│
│                                          │
│ 确定了：                                  │
│ • 首次创建项目体验作为 P0                  │
│ • 技术方案用 A，补弱网测试                 │
│                                          │
│ 没结论的：                                │
│ • 定制客户要不要接，等数据再定             │
│                                     [📌] │
└──────────────────────────────────────────┘
```

### 旧卡片自动收起

超过 15 分钟的卡片自动折叠为一行：

```
── 14:20 ~ 14:35 的 3 条内容（点击展开）──
```

保持信息流始终聚焦在最新内容。被 📌 钉住的卡片不会折叠。

## 用户互动

只有三种操作，从最轻到最重：

### 1. 无视（零成本）

大多数时候。AI 的卡片出来了，用户可能根本没看到，也可能扫了一眼。这就是最常见的"互动"。

设计要求：AI 必须接受被无视。不追问、不重复、不提醒。

### 2. 📌 钉住（一次点击）

"这个点有价值，我回头要看"。仅此而已。

钉住后的效果：
- 卡片不会被自动折叠
- 会议结束时，所有钉住的卡片会被收进保存的会议记录

不需要 👍👎、不需要评分、不需要分类。一个动作就够了。

### 3. 输入框提问（打字）

用户在输入框打字发送。AI 基于当前会议上下文回答。

输入框行为和现在基本一致，只有一个小改动：如果用户发了消息，AI 的回复卡片上会显示用户的原始问题（方便上下文回顾）。

## AI 行为控制

### 三档模式

顶栏的一个三档切换按钮，控制 AI 的主动程度：

| 模式 | AI 行为 | 典型场景 |
|------|---------|---------|
| **观察者** | 只在用户提问时回应。完全不主动输出。 | 不想被干扰，或会议内容敏感 |
| **顾问**（默认） | 有值得说的就说，但克制频率。同时用户可以随时提问。 | 日常会议 |
| **研究员** | 积极输出，频繁提问和发散联想。 | 头脑风暴、探索性讨论 |

切换方式：顶栏三个小圆点或 segmented control，一目了然。也可以在输入框直接说"安静一点"/"积极一点"来切换。

### "顾问"模式的频率控制

默认的"顾问"模式下，AI 的主动输出需要节制：

**输出节奏**：
- 不按固定时间间隔输出（现在的"每 N 条/每 N 秒"机制太机械）
- 改为：AI 每次分析后**自己判断**值不值得说。不值得就不说。
- 但设一个硬性下限：**两次主动输出之间至少间隔 2 分钟**，防止刷屏

**节点总结的触发**：
- AI 检测到话题明显切换时（语义信号：新关键词出现、之前的关键词消失）
- 会议结束信号（"就这样吧"、"差不多了"、录音停止）
- 30 分钟无总结时，强制一次

**闭嘴规则**：
- 会议内容是闲聊/寒暄 → 不说话
- 上一次输出用户完全没互动（没 📌、没追问）且没有新的高价值信息 → 降低输出频率
- 连续两次输出都没被理会 → 自动退到更安静的状态，直到有新的强信号

## AI 输出的内容标准

不是所有"分析结果"都值得输出。AI 应该在内部做一层过滤：

### 值得说的

- 会议中没人提到过的角度
- 对当前讨论方向的质疑或风险提醒
- 具体的、可操作的建议
- 对含混讨论的清晰化提炼（"你们争的其实是这个问题"）
- 用户概念不了解时的简要科普（仅在用户问时）

### 不值得说的

- 重复会议已经说过的内容（纯粹的"总结"）
- "可以考虑..."之类的模糊建议
- 显而易见的观点
- 对刚才内容的复述换一种说法

判断标准只有一个：**如果你是坐在旁边的人，这句话你会开口说吗？** 不会就别说。

## 会议结束后

### 自动保存

录音停止时，自动保存三个文件：

| 文件 | 内容 |
|------|------|
| `{时间}.mp3` | 录音 |
| `{时间}.txt` | 转写文本（已有） |
| `{时间}-ai.md` | AI 信息流全部卡片，📌 标记的排在最前 |

### AI 信息流保存格式

```markdown
# 会议记录 - 2026-03-06 14:00

## 📌 标记的要点

> 都倾向方案 A 了，但 WebSocket 在弱网下维护连接成本很高...

> **阶段小结**: 确定了 3 件事...

## 完整信息流

### 14:20
...

### 14:32
...
```

📌 标记的卡片提取到文件开头，方便散会后快速回顾。

## 对现有 UI 的具体改动清单

| 现有组件 | 改动 |
|---------|------|
| **顶栏** | 加三档模式切换（观察者/顾问/研究员） |
| **ChatView** | 气泡改卡片样式，加 📌 按钮，旧卡片自动折叠 |
| **ChatBubble** | 重命名为 InsightCard，去掉角色标签（系统/用户/AI），改为卡片形态 |
| **输入框** | 不变 |
| **"立即分析"按钮** | 保留，但改文案为"AI 说点什么"或直接去掉（顾问模式下 AI 会自己说） |
| **MeetingViewModel** | 触发逻辑从"定时/定量"改为"AI 自判断 + 节奏下限" |
| **保存逻辑** | 新增 AI 信息流的 Markdown 保存 |

## 不做的

- 不做卡片分类标签（💡🤔🔍——增加认知负担，AI 说的就是说的）
- 不做卡片的 👍👎（太多选择，📌 一个动作够了）
- 不做 AI 头像或人格化装饰（会议工具不需要拟人）
- 不做通知/弹窗/声音提醒（会议中任何打断都是灾难）
- 不做卡片拖拽、排序、分组（会议中没空做整理）
